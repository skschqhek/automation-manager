# automation-manager 제작 일대기

이 문서는 CurseForge 마인크래프트 서버 자동화, 절전 해제, 디스코드 공지, 디스코드 이모지 투표 기반 서버 실행 기능을 하나로 묶기까지의 과정과 시행착오를 정리한 기록입니다.

비밀번호, 봇 토큰, 웹후크 URL 같은 개인 정보는 이 문서에 포함하지 않습니다.

## 1. 어떻게 기능하는지

`automation-manager`는 Windows PowerShell 기반 자동화 묶음입니다. 바탕화면의 `automation-manager` 폴더 안에서 서버 실행, 서버 종료, 절전 해제, 디스코드 공지, 디스코드 반응 감지를 한 화면에서 관리합니다.

### 통합 UI

`AutomationManager.bat`를 실행하면 한글 UI가 열립니다. UI에서는 마인크래프트 서버 즉시 실행, 서버 시작/종료 예약, 예약 해제, 13:50 절전 해제 설정, 디스코드 메시지 테스트, 디스코드 반응 투표 메시지 게시, 설정 파일 열기 등을 수행할 수 있습니다.

### 마인크래프트 서버 자동화

`curseforge-server-automation` 폴더가 서버 자동 실행과 종료를 담당합니다. `Start-CurseForgeServer.ps1`은 설정된 서버 폴더에서 `run.bat`을 실행하고, 디스코드에 서버가 켜졌다는 공지와 부팅/종료 시각을 보냅니다. `Stop-CurseForgeServer.ps1`은 RCON으로 종료 전 안내 메시지를 보내고, 마지막에 `stop` 명령을 보내 서버를 종료합니다.

서버 종료 전 공지는 `config.json`의 `WarningMinutes`, `WarningMessageTemplate`, `StopNowMessage` 값으로 조정합니다.

### RCON

RCON은 마인크래프트 서버에 외부에서 명령을 보내는 통로입니다. 이 자동화에서는 종료 전 안내 메시지와 서버 종료 명령 전송에 사용됩니다. 서버의 `server.properties`에서 RCON이 켜져 있어야 하고, 자동화 설정 파일의 RCON 포트와 비밀번호가 맞아야 합니다.

### 13:50 절전 해제

`Send-Scheduled-Message` 폴더가 절전 해제를 담당합니다. `Install-Wake1350.ps1`은 Windows 작업 스케줄러에 13:50 절전 해제 작업을 등록하고, 절전 해제가 가능하도록 전원 설정을 조정합니다.

절전 해제 후 암호를 요구하지 않도록 `powercfg`의 `CONSOLELOCK` 값도 확인하고 변경했습니다.

### 디스코드 예약 메시지

`discord scheduled message` 폴더가 웹후크 메시지 발송을 담당합니다. 메시지는 매일 무조건 발송되는 것이 아니라, 오늘 서버 시작 예약이 있을 때만 발송됩니다.

현재는 다음 두 예약 중 하나라도 오늘 실행 예정이면 공지를 보냅니다.

- `OneButton Minecraft Server Start`
- `CurseForge Server Start`

### 디스코드 이모지 투표로 서버 실행

`discord-reaction-server-start` 폴더가 담당합니다.

흐름은 다음과 같습니다.

1. 봇이 지정 채널에 투표 메시지를 올립니다.
2. 봇이 기본 이모지를 한 번 붙입니다.
3. 감시 스크립트가 일정 간격으로 반응 수를 확인합니다.
4. 반응 수가 기준치에 도달하면 서버 시작 스크립트를 실행합니다.
5. 웹후크로 서버 시작 공지를 보냅니다.
6. 기준을 달성한 투표 메시지는 삭제합니다.
7. 새 투표를 올릴 때 이전 투표 메시지가 남아 있으면 먼저 삭제합니다.

`VoteStateFile`은 봇이 마지막으로 올린 투표 메시지 ID를 저장합니다. 그래서 사용자가 매번 `MessageId`를 직접 찾지 않아도 됩니다.

`StateFile`은 이미 기준 반응 수에 도달해 서버를 켰는지 기록합니다. `AllowRepeat`가 `false`이면 같은 투표로 서버를 여러 번 켜지 않도록 막습니다.

### 원버튼 예약 + 절전

루트의 `원버튼-예약-절전.ps1`과 `OneButton-Schedule-Sleep.ps1`이 담당합니다.

실행하면 가까운 13:50 기준으로 다음 작업을 예약합니다.

- 13:50 컴퓨터 절전 해제
- 서버 시작 10분 전 디스코드 공지 발송
- 설정된 마인크래프트 서버 시작 시각에 서버 실행

예약을 등록한 뒤 10초 대기 후 컴퓨터를 절전 모드로 전환합니다.

## 2. 기능하기 위해 거쳐온 방법

처음 목표는 CurseForge 마인크래프트 서버를 정해진 시간에 자동으로 켜고 끄는 것이었습니다. 이후 서버가 꺼지기 전에 유저에게 안내가 필요해 RCON 기반 공지를 추가했습니다.

그 다음 서버 자동 실행 실패 원인을 추적했습니다. 핵심은 한글이 포함된 바탕화면 경로와 실제 서버 폴더명이 정확히 일치하지 않는 문제였습니다. 이를 해결하기 위해 `{Desktop}` 치환 방식을 도입했습니다.

이후 컴퓨터가 절전 상태여도 서버 실행 시간 전에 준비되도록 13:50 절전 해제 스크립트를 만들었습니다. 절전 해제 후 로그인 화면에서 자동화가 막히지 않도록 암호 요구 설정도 함께 확인했습니다.

그 다음 디스코드 공지 기능을 붙였습니다. 처음에는 정해진 시간마다 메시지를 보내는 방식이었지만, 불필요한 공지를 막기 위해 오늘 서버 실행 예약이 있을 때만 발송되도록 바꿨습니다.

여러 자동화 폴더가 흩어져 관리가 어려워진 뒤에는 `automation-manager`로 통합했습니다. UI를 한글화하고, 서버/절전/디스코드/반응 감지를 한 화면에서 관리하게 만들었습니다.

마지막으로 이모지 반응 수가 기준에 도달하면 서버를 켜는 기능을 추가했습니다. 처음에는 사용자가 메시지 ID를 직접 넣어야 했지만, 이후 봇이 스스로 투표 메시지를 올리고 그 메시지를 감시하도록 개선했습니다.

## 3. 시행착오

### 한글 경로 문제

`바탕 화면` 같은 한글 경로가 PowerShell 화면에서 깨져 보이면서 실제 폴더를 찾지 못하는 일이 있었습니다. 해결책은 경로를 직접 쓰는 대신 `{Desktop}` 치환값을 사용해 Windows가 바탕화면 경로를 직접 찾게 하는 것이었습니다.

### `ServerDir`가 맞아 보이는데도 실패한 문제

눈으로 보기에는 경로가 맞아 보여도 실제 문자열의 한글 깨짐, 공백, 쉼표, 폴더명 차이 때문에 `Test-Path`가 `False`를 반환했습니다. 실제 폴더 목록을 확인하고 서버 폴더명을 정확히 맞춰 해결했습니다.

### PowerShell 실행 정책

스크립트 실행이 실행 정책 때문에 막힐 수 있었습니다. 그래서 실행 명령에 `-ExecutionPolicy Bypass`를 사용했습니다. UI에서 실행하는 스크립트도 같은 방식으로 실행됩니다.

### PowerShell 프로필 경고

PowerShell 시작 시 사용자 프로필 스크립트가 실행 정책 때문에 막힌다는 경고가 보였습니다. 자동화 스크립트는 `-NoProfile` 옵션으로 실행되므로 기능에는 영향을 주지 않게 했습니다.

### Discord API `40333 internal network error`

봇 권한과 채널 권한을 맞췄는데도 `Invoke-RestMethod`에서 `40333 internal network error`가 발생했습니다. 봇 토큰과 채널 접근이 정상임을 확인한 뒤, Discord API 호출을 `.NET HttpClient` 방식으로 바꿔 해결했습니다.

### 디스코드 권한 혼란

OAuth2 화면과 Bot 화면 중 어디서 권한을 만져야 하는지 혼란이 있었습니다. 최종적으로는 봇 초대 URL에서 필요한 권한을 포함하고, 서버 및 채널 개별 권한에서도 봇 역할에 권한을 주는 방식으로 정리했습니다.

필요 권한은 채널 보기, 메시지 보내기, 메시지 기록 보기, 반응 추가, 메시지 관리입니다.

### `ChannelId`, `MessageId` 찾기 문제

개발자 모드를 켜도 ID를 찾기 어렵다는 문제가 있었습니다. 그래서 봇이 직접 투표 메시지를 올리고, 그 메시지 ID를 `VoteStateFile`에 저장하도록 바꿨습니다.

### 이전 투표 메시지 정리

이전 투표 메시지가 남으면 다음 투표와 혼동될 수 있었습니다. 현재는 반응 수가 기준에 도달하거나 새 투표를 올릴 때 이전 투표 메시지를 삭제합니다.

### 상태 파일 문제

`StateFile`이 남아 있으면 `AllowRepeat: false`에서 다음 실행을 막을 수 있다는 우려가 있었습니다. 현재는 투표 메시지 상태와 실행 달성 상태를 분리했고, 필요하면 상태 초기화 기능으로 지울 수 있습니다.

### 원버튼 래퍼 불안정성

한글 파일명 실행 문제를 피하려고 ASCII 이름의 래퍼를 만들었습니다. 처음에는 첫 번째 PowerShell 파일을 실행하는 방식이라 불안정했지만, 이후 원버튼 예약/절전 스크립트만 찾도록 조건을 좁혔습니다.

### 지나간 1회성 예약 작업

원버튼 예약은 1회성 작업을 만들기 때문에 실행 후 `NextRunTime`이 비어 있는 잔여 작업이 남을 수 있습니다. 기능을 방해하지는 않지만 UI에서 혼란을 줄 수 있어 필요하면 예약 해제로 정리할 수 있습니다.

## 현재 점검 결과

마지막 점검 기준으로 다음을 확인했습니다.

- 전체 PowerShell 스크립트 문법 정상
- 전체 JSON 설정 파일 파싱 정상
- 디스코드 API 호출에서 `Invoke-RestMethod` 제거
- 주요 예약 작업이 실제 존재하는 스크립트 파일을 가리킴
- 서버 시작 예약과 종료 예약이 작업 스케줄러에 등록됨
- 불필요한 디스코드 투표 상태 파일이 남아 있지 않음

## GitHub 게시 시 주의 사항

실제 `config.json` 파일은 GitHub에 올리면 안 됩니다. 봇 토큰, 웹후크 URL, RCON 비밀번호 같은 개인 정보가 들어 있을 수 있기 때문입니다.

공개 저장소에는 `config.example.json`만 올리고 실제 `config.json`, 상태 파일, 로그 파일은 `.gitignore`로 제외해야 합니다.
